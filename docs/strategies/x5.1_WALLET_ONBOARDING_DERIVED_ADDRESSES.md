# MS5 Phase 1 Strategy - Wallet Onboarding & Derived Addresses

Status: Complete (retrospective reconstruction).
Parent milestone rollup: [`docs/PLAN.md`](../PLAN.md)
Historical execution window: 2025-11-07 through 2025-11-08.
Current UX requirements: [`docs/specs/WALLET_XPUB_UX_SPEC.md`](../specs/WALLET_XPUB_UX_SPEC.md)

> This checklist was reconstructed on 2026-07-17; it was not the prospective checklist used during MS5. Checkmarks mean the repository history records the work as shipped. MS5 had no recorded internal phase structure, so `.1` is used only to fit the current strategy-doc naming convention.

Move invoice payment destinations from manually entered addresses to watch-only wallet onboarding and a unique BIP84-derived receive address for each invoice.

## Historical boundary

MS5 shipped the first cursor model: one mutable `next_derivation_index` attached to the current user wallet key, reset when that key changed. It did not preserve per-key lineage or historical cursors. MS14 later replaced that assumption with key-aware lineage/cursor safety. The current wallet UX spec also includes substantial onboarding and validation work that postdates MS5.
