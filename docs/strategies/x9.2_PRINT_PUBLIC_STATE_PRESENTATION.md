# MS9 Phase 2 Strategy - Print & Public State Presentation

Status: Complete (retrospective reconstruction).
Parent milestone: [`docs/milestones/x9_PRINT_PUBLIC_POLISH.md`](../milestones/x9_PRINT_PUBLIC_POLISH.md)
Historical execution date: 2025-11-18.

> This phase boundary was reconstructed on 2026-07-18 around the active/disabled print and public presentation changes in `2372100`. It was not named or checked off as Phase 9.2 during original execution.

## Goal

Present active public invoices as payable artifacts while turning disabled or expired links into safe, friendly unavailable states.

## Reconstruction basis

Primary evidence: the status/layout requirements in `0428384`, implementation commit `2372100`, and the public-share tests updated by that commit.

## 1. Resolve public availability before rendering payment content

1. [x] Load the invoice by public token without exposing owner-only routes.
2. [x] Evaluate both enabled state and optional expiry.
3. [x] Keep active and unavailable presentation in the public print path.
4. [x] Preserve crawler-blocking behavior for public invoice output.

## 2. Render the active invoice state

1. [x] Show the effective billing identity and contact details.
2. [x] Show invoice metadata and client-facing payment status.
3. [x] Keep Expected/Received/Outstanding values readable.
4. [x] Keep the QR, BIP21 intent, and rate-as-of guidance together.
5. [x] Preserve payment history where the current invoice state warrants it.

## 3. Render the unavailable state safely

1. [x] Replace payment instructions with a friendly disabled/expired message.
2. [x] Reveal no payment address, QR, amount, or payment-history detail.
3. [x] Include a safe contact path from the resolved billing identity when available.
4. [x] Keep the page visually related to the active invoice without implying it remains payable.

## 4. Align print presentation

1. [x] Keep private print independent from public-link availability.
2. [x] Render the same billing identity and footer as active public output.
3. [x] Keep paid versus outstanding messaging visible.
4. [x] Preserve paper-friendly payment and summary layout.

## 5. Verify public behavior

1. [x] Update public-share assertions for the new unavailable-state response.
2. [x] Prove active public output renders branding details.
3. [x] Preserve noindex and private-print separation tests.
4. [x] Keep disabled/expired output free of active payment details.

## Phase Exit Criteria

- [x] Active public links present a complete payable invoice.
- [x] Disabled and expired links present a safe unavailable state.
- [x] Public unavailability hides payment details.
- [x] Private print remains available to the authenticated owner.
- [x] Public-share tests cover active and unavailable rendering.

## Historical boundary

MS9 polished the existing public-print branch. MS13 later unified private/public content through a shared template and expanded explicit public-state coverage; this phase records the earlier presentation boundary.
