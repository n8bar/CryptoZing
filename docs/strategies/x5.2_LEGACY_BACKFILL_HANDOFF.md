# MS5 Phase 2 Strategy - Legacy Backfill & Handoff

Status: Complete (retrospective reconstruction).
Parent milestone: [`docs/milestones/x5_WALLET_ONBOARDING_DERIVED_ADDRESSES.md`](../milestones/x5_WALLET_ONBOARDING_DERIVED_ADDRESSES.md)
Historical execution date: 2025-11-08.

> This phase boundary was reconstructed on 2026-07-18 around the migration/tooling handoff in `19a05bf`. It was not named or checked off as Phase 5.2 during original execution.

## Goal

Move invoices created before runtime derivation onto the new watch-only address model with inspectable, deterministic backfill tooling.

## Reconstruction basis

Primary evidence: commit `19a05bf`, which added the command, seed helpers, derivation-script rename, and historical PLAN update.

## 1. Add the legacy assignment command

1. [x] Add `wallet:assign-invoice-addresses`.
2. [x] Select users with their wallet settings.
3. [x] Report users that have invoice data but no wallet configuration.
4. [x] Select only invoices whose `payment_address` is missing.
5. [x] Process invoices in stable ascending ID order per user.

## 2. Derive and persist legacy addresses

1. [x] Use the same `HdWallet` service as runtime invoice creation.
2. [x] Derive from the configured public key, network, and current cursor.
3. [x] Report the invoice, derived address, and index being assigned.
4. [x] Inside a transaction:
   1. [x] persist `payment_address`
   2. [x] persist `derivation_index`
   3. [x] advance `next_derivation_index`
5. [x] Continue through all eligible invoices/users.

## 3. Add preview and fixture support

1. [x] Add `--dry-run` so the command reports changes without writing.
2. [x] Advance the in-memory cursor during dry-run so previewed addresses/indexes match execution order.
3. [x] Report the total invoices changed or proposed.
4. [x] Add public-extended-key-only seed helpers for local test data.
5. [x] Rename the Node helper to `derive-address.cjs` for the package/module setup.

## 4. Close the milestone

1. [x] Update historical PLAN with the completed wallet onboarding/derivation outcome.
2. [x] Record the command and `/wallet/settings` delivery in the changelog.
3. [x] Leave newly created and legacy invoices on the same address/index model.
4. [x] Hand the derived invoice addresses to MS6 blockchain detection.

## Phase Exit Criteria

- [x] Legacy invoices without payment addresses can be identified deterministically.
- [x] Dry-run previews the exact assignment order without database writes.
- [x] Real execution persists address/index and advances the shared cursor.
- [x] Tracked fixtures contain public derivation material only.
- [x] All invoice generations now converge on the derived-address model.

## Historical boundary

This command was the first migration path and inherited MS5's one-key/one-cursor assumptions. Later correction and key-lineage tooling superseded those assumptions; this doc records the original handoff rather than the current operational command surface.
