# MS18 Phase 2 Strategy — Content Audit & Production

Status: Active.
Parent milestone doc: [`docs/milestones/18_PRERELEASE_CONTENT_SEO.md`](../milestones/18_PRERELEASE_CONTENT_SEO.md)

**Goal:** Audit the existing Helpful Notes, make Adapt/Inspire/Skip decisions, and produce all planned content — moving directly from audit decision to writing for each piece.

## Content workflow

1. **Author** — create `site/staging/<slug>.md` with front matter: `layout: article.njk`, `title`, `description`, `canonical: https://cryptozing.app/learn/<slug>/` (the future production URL)
2. **Build** — `cd site && npx @11ty/eleventy` — or run `npx @11ty/eleventy --watch` once and skip this step on every subsequent save
3. **Stage** — review locally at `http://<dev-server-LAN>/staging/<slug>/` via Sail (no push needed), or push to `main` for a live shareable staging URL at `https://cryptozing.app/staging/<slug>/` — noindexed, not linked from anywhere
4. **Review** — read it, revise, repeat until satisfied; update [`docs/CONTENT_PROMISES.md`](../CONTENT_PROMISES.md) with any new promises
5. **Publish** — move file from `site/staging/<slug>.md` to `site/learn/<slug>.md`; push to `main`
6. **Verify** — confirm HTTP 200 at `https://cryptozing.app/learn/<slug>/`
7. **Sitemap** — add the URL to `site/sitemap.xml` with today's date as `lastmod`
