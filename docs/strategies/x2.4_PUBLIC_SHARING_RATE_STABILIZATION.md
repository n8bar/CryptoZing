# MS2 Phase 4 Strategy - Public Sharing & Rate Stabilization

Status: Complete (retrospective reconstruction).
Parent milestone: [`docs/milestones/x2_INVOICE_UX_FOUNDATIONS.md`](../milestones/x2_INVOICE_UX_FOUNDATIONS.md)
Historical execution window: 2025-10-30 through 2025-11-04.

> This phase boundary was reconstructed on 2026-07-18 from the public-sharing implementation and its immediate rate/display corrections. It was not named or checked off as Phase 2.4 during original execution.

## Goal

Make invoices safely shareable through controlled public links and stabilize the first cached/live rate presentation across authenticated and public payment output.

## Reconstruction basis

Primary evidence: `a5dbca5`, `c74722e`, combined PR #4 merge `4f40d97`, `23205a2` (PR #6), `9fbb216` (PR #10), and `5208f6b` (PR #11).

## 1. Add live-rate retrieval

1. [x] Add the first live Coinbase BTC/USD spot fetch.
2. [x] Retain cached-rate support for normal invoice display.
3. [x] Recompute displayed BTC from the applicable rate without replacing the invoice's stored USD amount.

## 2. Add controlled public sharing

1. [x] Add public token, enabled-state, and expiry fields to invoices.
2. [x] Add owner actions to enable and disable a public link.
3. [x] Add token rotation/regeneration.
4. [x] Add expiry presets/handling.
5. [x] Add `/p/{token}` public invoice output based on the print surface.
6. [x] Attach the public noindex/noarchive response header.
7. [x] Prevent disabled or expired shares from exposing invoice content.

## 3. Stabilize rate presentation

1. [x] Add a visible "Rate as of" timestamp on invoice show.
2. [x] Add explicit manual refresh while preserving cached normal reads.
3. [x] Correct refresh so both displayed rate-derived values and timestamp update.
4. [x] Make show-page BTC and QR output derive from the cached/current display rate.
5. [x] Keep public output on a fresh-rate-first path with fallback.

## 4. Merge and verify the foundation

1. [x] Merge the combined client/invoice UI through PR #4.
2. [x] Land the rate-as-of/refresh enhancement through PR #6.
3. [x] Land the refresh correction through PR #10.
4. [x] Land dynamic cached-rate BTC/QR behavior through PR #11.
5. [x] Record the resulting public/share and rate feature set in historical PLAN.

## Phase Exit Criteria

- [x] Owners can enable, disable, rotate, and expire public invoice links.
- [x] Public output does not expose disabled/expired invoice content and carries crawler-blocking headers when active.
- [x] Authenticated invoice output shows cached rate timing and supports explicit refresh.
- [x] BTC and QR display values respond to the current display rate.

## Historical boundary

This phase made the rate UX usable; it did not yet establish the final rate/currency invariants. MS3 added focused public/rate tests the next day, and MS4 then codified USD canonicality, satoshi precision, TTL behavior, and shared formatting.
