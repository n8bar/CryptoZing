# MS3 Phase 1 Strategy - Test Hardening

Status: Complete (retrospective reconstruction).
Parent milestone rollup: [`docs/PLAN.md`](../PLAN.md)
Historical execution date: 2025-11-07.
Surviving checklist: [`docs/qa/tests/TEST_HARDENING.md`](../qa/tests/TEST_HARDENING.md)

> This strategy was reconstructed on 2026-07-17 from the surviving test draft and the commits that implemented it. MS3 had no recorded internal phase structure, so `.1` is used only to fit the current strategy-doc naming convention.

Add baseline feature coverage around the highest-risk behaviors established by MS1 and MS2: public links, rate refresh/cache behavior, QR/BIP21 output, and soft-delete recovery.

## Reconstruction basis

The ordered test draft landed in commit `b67ee67` (PR #16). Its implementation is recorded by commits `2cf23cb`, `110a487`, `aa39b22` (PR #19), and `110f6d4` (PR #20), followed by the completed-milestone summary in historical PLAN commit `a961df0` and the matching 2025-11-07 changelog entry.

## 1. Define the hardening scenarios

1. [x] Write explicit preconditions, request steps, and assertions before implementing the new feature tests.
2. [x] Cover public-share enable, disable, token rotation, expiry, and unauthorized toggling.
3. [x] Cover public-page crawler directives without applying noindex metadata to private print views.
4. [x] Cover cached-rate reuse, explicit refresh, updated timestamps, and rate-fetch failure behavior.
5. [x] Cover BIP21 and QR output on invoice show and print views.
6. [x] Cover client/invoice trash listing, restore, and owner-only permanent deletion.

## 2. Implement the public-share tests

1. [x] Prove the record changes and flash responses for enabling and disabling a public link.
2. [x] Prove rotating a link changes its token while leaving sharing active.
3. [x] Prove another user cannot toggle an invoice's public sharing.
4. [x] Prove an active public invoice returns the expected noindex header.
5. [x] Prove private print output does not inherit public noindex metadata.
6. [x] Prove an expired public token no longer returns invoice content.

## 3. Implement rate and payment-output tests

1. [x] Prove invoice show reuses a cached rate without a network call.
2. [x] Prove explicit refresh fetches a new rate and changes rendered output.
3. [x] Prove a rate-fetch failure leaves the page usable.
4. [x] Prove invoice show renders the expected BIP21 link and QR/copy controls.
5. [x] Prove print output includes the QR and wallet prompt.

## 4. Implement trash-flow tests

1. [x] Prove deleted clients appear in trash and can be restored.
2. [x] Prove deleted invoices appear in trash and can be restored.
3. [x] Prove a non-owner cannot permanently delete another user's client.
4. [x] Prove a non-owner cannot permanently delete another user's invoice.

## 5. Verify MS3

1. [x] Keep the test cases deterministic with factories, authenticated requests, faked time where needed, and mocked rate/network behavior.
2. [x] Land the feature tests in the named test files recorded by git history.
3. [x] Update PLAN and the changelog to mark the hardening suite complete.

## Historical boundary

This was a targeted baseline, not a claim of exhaustive application coverage. Later milestones expanded and reshaped these tests as payment, wallet, notification, and UX behavior evolved. The surviving test draft remains useful as the prospective checklist; this strategy records which parts the November 7 commits actually implemented.
